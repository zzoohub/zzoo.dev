## 콘텐츠 아키텍처

프로젝트마다 프로덕트 브리프, 케이스 스터디, 설계 문서의 세 가지 문서 레이어가 있습니다. 모든 프론트매터는 Zod 스키마로 검증됩니다. 이미지 경로는 정제되고, 슬러그는 타입 체크됩니다.

콘텐츠는 로케일별 MDX 파일로 관리됩니다 — 6개 언어(EN, ES, PT-BR, ID, JA, KO), 번역이 없는 경우 자동 폴백.

## 검색 가시성

세 가지 발견 채널을 타겟합니다:

- **SEO** — 정적 생성, 시맨틱 HTML, hreflang 대체 링크가 포함된 XML 사이트맵, 캐노니컬 URL. Lighthouse 95+.
- **AEO** — AI 답변 엔진을 위한 JSON-LD 스키마(`Person`, `WebSite`, `BlogPosting`, `SoftwareApplication`, `BreadcrumbList`). 콘텐츠 인덱싱을 위한 RSS 피드.
- **GEO** — 생성형 AI 엔진을 위한 `llms.txt`와 깔끔한 MDX 콘텐츠.

## 오픈소스

전체 코드베이스가 [오픈소스](https://github.com/zzoohub/zzoo.dev)입니다. Next.js 16, React 19, TypeScript, Tailwind CSS v4로 구축, Cloudflare Workers에 배포.
