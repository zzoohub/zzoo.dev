구글이 포트폴리오를 인덱싱한다. Perplexity가 답변에 인용한다. ChatGPT가 누군가 기술 스택을 물었을 때 정보를 뽑아간다. 아니면 아무 일도 일어나지 않는다 — 사이트가 기계는 파싱할 수 없는 시각적 브로슈어에 불과하니까.

개발자 포트폴리오 대부분은 독자 하나만 최적화한다: 사람. 이 사이트는 세 부류를 동시에 서빙하는 실험이다 — 평가하는 사람, 순위를 매기는 검색 엔진, 추출하고 인용하는 AI 시스템. 순수 정적 HTML로.

## 세 독자, 하나의 제약

제약은 런타임 제로다. 서버 없음, 데이터베이스 없음, API 라우트 없음. 모든 페이지가 사전 렌더링된 HTML이고 Vercel 엣지에서 서빙된다. 제한적으로 들리지만, 이게 만드는 것들이 있다: 1초 미만 로딩, 공격 면적 제로, `main`에 push하면 끝나는 배포.

정적 HTML이 사람의 가독성을 해치지 않으면서 기계에 충분한 구조를 담을 수 있을까. JSON-LD 스키마가 답변 엔진에 모든 엔티티의 맥락을 전달한다. 사이트 루트의 `llms.txt`가 LLM에 사이트 지도를 제공한다. 시맨틱 HTML과 hreflang 대체 링크가 전통 검색을 처리한다. 세 채널이 같은 콘텐츠를 공유하고, 읽는 방식만 다르다.

## 사고 과정을 보여주는 콘텐츠

프로젝트마다 탭 세 개. Overview는 5초 스캔용 — 이것이 뭔지, 왜 중요한지. Design은 트레이드오프와 기각된 대안을 원하는 프로덕트 씽커용. Engineering은 아키텍처 결정과 구현이 궁금한 빌더용.

채용 담당자는 탭 하나를 읽는다. 동료 엔지니어는 셋 다 읽는다. 누구도 다른 사람을 위한 콘텐츠를 거쳐갈 필요가 없다.

모든 콘텐츠가 Zod 검증 프론트매터를 가진 파일 기반 MDX다. 한국어 번역이 없으면 영어로 폴백한다. CMS도 데이터베이스도 없다 — git 저장소의 파일이 전부, 코드처럼 버전 관리된다.

## 번역이 아니라 병렬 콘텐츠

영어와 한국어가 병렬로 존재한다. 구조는 같되 각 언어로 자연스럽게 쓴다. 언어 전환은 버튼 하나 — 로케일이 두 개인데 드롭다운을 쓸 이유가 없다. 새 콘텐츠는 한쪽 언어로 먼저 배포해도 폴백 덕분에 깨지지 않는다.

## 오픈소스

[코드베이스 전체](https://github.com/zzoohub/zzoo.dev)가 공개되어 있다. 가져갈 만한 건 콘텐츠 아키텍처다 — 세 탭 프로젝트 시스템, Zod 스키마, 세 채널 발견. 포크해서 콘텐츠를 바꾸고 배포하면 된다.
