Al construir este sitio, dos cosas fueron las más importantes. Primero, no solo humanos sino también sistemas de IA deberían poder leer y entender este sitio. Segundo, debería ser accesible de forma natural para usuarios en cualquier idioma. Estas dos consideraciones definieron la estructura y las decisiones técnicas del sitio.

## Arquitectura de contenido

Cada proyecto tiene tres capas de documentos: brief de producto, caso de estudio y documento de diseño. No solo una lista de funcionalidades, sino una estructura que captura contexto y toma de decisiones.

Todo el frontmatter se valida con schemas Zod. El contenido se gestiona como archivos MDX divididos por locale — 6 idiomas (EN, ES, PT-BR, ID, JA, KO) con fallback automático cuando falta una traducción.

## Visibilidad en búsquedas

El sitio apunta a tres canales de descubrimiento — búsqueda en Google, motores de respuesta de IA e IA generativa — para que el contenido se encuentre donde sea que la gente busque:

- **SEO** — Generación estática, HTML semántico, sitemap XML con alternates hreflang, URLs canónicas. Lighthouse 95+.
- **AEO** — Schemas JSON-LD (`Person`, `WebSite`, `BlogPosting`, `SoftwareApplication`, `BreadcrumbList`) para motores de respuesta de IA. Feed RSS para indexación de contenido.
- **GEO** — `llms.txt` y contenido MDX limpio para motores de IA generativa.

## Open source

Todo el codebase es [open source](https://github.com/zzoohub/zzoo.dev). Construido con Next.js 16, React 19, TypeScript, Tailwind CSS v4, deployado en Cloudflare Workers. Cualquiera puede hacer fork, adaptar la estructura de contenido y hacerlo suyo.
