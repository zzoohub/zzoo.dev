## Arquitectura de contenido

Cada proyecto tiene tres capas de documentos: brief de producto, caso de estudio y documento de diseño. Todo el frontmatter se valida con schemas Zod. Las rutas de imágenes están sanitizadas. Los slugs tienen verificación de tipos.

El contenido se gestiona como archivos MDX divididos por locale — 6 idiomas (EN, ES, PT-BR, ID, JA, KO) con fallback automático cuando falta una traducción.

## Visibilidad en búsquedas

El sitio apunta a tres canales de descubrimiento:

- **SEO** — Generación estática, HTML semántico, sitemap XML con alternates hreflang, URLs canónicas. Lighthouse 95+.
- **AEO** — Schemas JSON-LD (`Person`, `WebSite`, `BlogPosting`, `SoftwareApplication`, `BreadcrumbList`) para motores de respuesta de IA. Feed RSS para indexación de contenido.
- **GEO** — `llms.txt` y contenido MDX limpio para motores de IA generativa.

## Open source

Todo el codebase es [open source](https://github.com/zzoohub/zzoo.dev). Construido con Next.js 16, React 19, TypeScript, Tailwind CSS v4, deployado en Cloudflare Workers.
