## El sitio es el caso de estudio

La mayoría de portafolios de desarrollador muestran proyectos. Este es el proyecto.

zzoo.dev está construido con el mismo flujo de trabajo de IA descrito en el blog — 29 skills, 7 agentes, 3 departamentos operando a través de una [constitución CLAUDE.md](/es/blog/my-claude-code-setup). Cada página que estás leyendo fue producida, auditada y verificada por ese sistema. El sitio no es solo un portafolio. Es la prueba viva de que una persona puede lanzar un producto multilingüe de nivel producción usando desarrollo impulsado por IA.

## Qué lo hace diferente

**Arquitectura de contenido, no un template.** Cada proyecto en este portafolio tiene tres capas de documentos: un brief de producto, un caso de estudio y un documento de diseño. Todo el frontmatter se valida con schemas Zod. Las rutas de imágenes están sanitizadas. Los slugs tienen verificación de tipos. No es un template de blog con una página de proyectos añadida — es un sistema de contenido diseñado para la profundidad.

**Seis idiomas, no dos.** Cada pieza de contenido existe en inglés, español, portugués, indonesio, japonés y coreano — con fallback automático de locale cuando falta una traducción. No es un feature flag. Es un pipeline de contenido que escala.

**Construido para tres canales de descubrimiento.** SEO te indexa. AEO (schemas JSON-LD para `Person`, `WebSite`, `BlogPosting`, `SoftwareApplication`) te cita en motores de respuesta. GEO (`llms.txt` + MDX limpio) te representa con precisión en IA generativa. La mayoría de portafolios manejan uno de estos. Este maneja los tres.

## Open source

Todo el codebase es open source. Clónalo, estudia la arquitectura de contenido, adapta el flujo de trabajo de IA. El valor real no es el template — es el sistema detrás.
