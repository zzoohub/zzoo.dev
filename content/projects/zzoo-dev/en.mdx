Building this site, two things mattered most. First, not just humans but also AI systems should be able to read and understand this site. Second, it should be naturally accessible to users in any language. These two considerations shaped the site's structure and technical choices.

## Content architecture

Each project has three document layers: product brief, case study, and design doc. Not just a feature list, but a structure that captures context and decision-making.

All frontmatter is validated through Zod schemas. Content is managed as MDX files split by locale — English and Korean with automatic fallback when a translation is missing.

## Search visibility

The site targets three discovery channels — Google search, AI answer engines, and generative AI — so content can be found wherever people look:

- **SEO** — Static generation, semantic HTML, XML sitemap with hreflang alternates, canonical URLs. Lighthouse 95+.
- **AEO** — JSON-LD schemas (`Person`, `WebSite`, `BlogPosting`, `SoftwareApplication`, `BreadcrumbList`) for AI answer engines. RSS feed for content indexing.
- **GEO** — `llms.txt` and clean MDX content for generative AI engines.

## Open source

The entire codebase is [open source](https://github.com/zzoohub/zzoo.dev). Built with Next.js 16, React 19, TypeScript, Tailwind CSS v4, deployed to Vercel. Anyone can fork it, adapt the content structure, and make it their own.
