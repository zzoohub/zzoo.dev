Ao construir este site, duas coisas foram as mais importantes. Primeiro, não apenas humanos mas também sistemas de IA deveriam conseguir ler e entender este site. Segundo, deveria ser naturalmente acessível para usuários em qualquer idioma. Essas duas considerações moldaram a estrutura e as escolhas técnicas do site.

## Arquitetura de conteúdo

Cada projeto tem três camadas de documentos: product brief, estudo de caso e design doc. Não apenas uma lista de funcionalidades, mas uma estrutura que captura contexto e tomada de decisão.

Todo o frontmatter é validado por schemas Zod. O conteúdo é gerenciado como arquivos MDX divididos por locale — 6 idiomas (EN, ES, PT-BR, ID, JA, KO) com fallback automático quando uma tradução está faltando.

## Visibilidade em busca

O site mira três canais de descoberta — busca no Google, motores de resposta de IA e IA generativa — para que o conteúdo seja encontrado onde quer que as pessoas procurem:

- **SEO** — Geração estática, HTML semântico, sitemap XML com hreflang alternates, URLs canônicas. Lighthouse 95+.
- **AEO** — Schemas JSON-LD (`Person`, `WebSite`, `BlogPosting`, `SoftwareApplication`, `BreadcrumbList`) para motores de resposta por IA. Feed RSS para indexação de conteúdo.
- **GEO** — `llms.txt` e conteúdo MDX limpo para motores de IA generativa.

## Open source

A codebase inteira é [open source](https://github.com/zzoohub/zzoo.dev). Construída com Next.js 16, React 19, TypeScript, Tailwind CSS v4, deployada na Vercel. Qualquer pessoa pode fazer fork, adaptar a estrutura de conteúdo e fazer do jeito dela.
