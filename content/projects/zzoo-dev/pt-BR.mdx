## O site é o estudo de caso

A maioria dos portfólios de desenvolvedor mostram projetos. Este é o projeto.

zzoo.dev é construído pelo mesmo workflow de IA descrito no blog — 29 skills, 7 agentes, 3 departamentos operando através de uma [constituição CLAUDE.md](/pt-BR/blog/my-claude-code-setup). Cada página que você está lendo foi produzida, auditada e verificada por esse sistema. O site não é apenas um portfólio. É a prova viva de que uma pessoa pode entregar um produto multilíngue de nível produção usando desenvolvimento orientado por IA.

## O que torna diferente

**Arquitetura de conteúdo, não um template.** Cada projeto neste portfólio tem três camadas de documentos: um product brief, um estudo de caso e um design doc. Todo o frontmatter é validado por schemas Zod. Caminhos de imagem são sanitizados. Slugs têm verificação de tipo. Não é um template de blog com uma página de projetos acoplada — é um sistema de conteúdo projetado para profundidade.

**Seis idiomas, não dois.** Cada peça de conteúdo existe em inglês, espanhol, português, indonésio, japonês e coreano — com fallback automático de locale quando uma tradução está faltando. Não é um feature flag. É um pipeline de conteúdo que escala.

**Construído para três canais de descoberta.** SEO te indexa. AEO (schemas JSON-LD para `Person`, `WebSite`, `BlogPosting`, `SoftwareApplication`) te cita em motores de resposta. GEO (`llms.txt` + MDX limpo) te representa com precisão em IA generativa. A maioria dos portfólios lida com um desses. Este lida com os três.

## Open source

A codebase inteira é open source. Clone, estude a arquitetura de conteúdo, adapte o workflow de IA. O valor real não é o template — é o sistema por trás.
