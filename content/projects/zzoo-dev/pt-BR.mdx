## Arquitetura de conteúdo

Cada projeto tem três camadas de documentos: product brief, estudo de caso e design doc. Todo o frontmatter é validado por schemas Zod. Caminhos de imagem são sanitizados. Slugs têm verificação de tipo.

O conteúdo é gerenciado como arquivos MDX divididos por locale — 6 idiomas (EN, ES, PT-BR, ID, JA, KO) com fallback automático quando uma tradução está faltando.

## Visibilidade em busca

O site mira três canais de descoberta:

- **SEO** — Geração estática, HTML semântico, sitemap XML com hreflang alternates, URLs canônicas. Lighthouse 95+.
- **AEO** — Schemas JSON-LD (`Person`, `WebSite`, `BlogPosting`, `SoftwareApplication`, `BreadcrumbList`) para motores de resposta por IA. Feed RSS para indexação de conteúdo.
- **GEO** — `llms.txt` e conteúdo MDX limpo para motores de IA generativa.

## Open source

A codebase inteira é [open source](https://github.com/zzoohub/zzoo.dev). Construída com Next.js 16, React 19, TypeScript, Tailwind CSS v4, deployada na Cloudflare Workers.
