순수 정적 사이트입니다. 서버 사이드 렌더링 없음, API 라우트 없음, 데이터베이스 없음, 어떤 종류의 백엔드도 없음. 모든 페이지가 빌드 타임에 사전 렌더링되어 Vercel의 글로벌 엣지 네트워크에서 정적 HTML로 서빙됩니다. 이 아키텍처 제약은 의도적입니다: 최소한의 공격 면, 거의 즉각적인 로딩 시간, 그리고 "main에 push하고 잊어버려도 되는" 배포 모델.

## 아키텍처: SSG 전용

사이트 전체가 Next.js 16의 App Router를 통해 정적 생성됩니다. 빌드 타임에 `generateStaticParams()`가 모든 로케일과 slug 조합을 사전 렌더링합니다. 빌드 출력에서 모든 라우트가 `●`(static) — `ƒ`(dynamic)는 절대 없음.

왜 SSR이 아니라 SSG인가? 콘텐츠가 변경되는 시점은 사용자가 방문할 때가 아니라 개발자가 커밋을 push할 때입니다. 사용자별 데이터, 인증, 폼 제출이 없습니다. SSR은 이 유스케이스에서 이점 없이 복잡성만 추가할 것입니다 (콜드 스타트, 서버 비용, 캐싱 전략).

트레이드오프: 콘텐츠 변경 때마다 리빌드가 필요합니다. 하지만 Vercel의 GitHub 연동으로 `main`에 push하면 2분 이내에 자동 빌드와 배포가 됩니다.

## Server Components 우선

React 19 Server Components가 기본입니다. 대부분의 페이지가 프레임워크 런타임 외에 클라이언트 사이드 JavaScript를 전혀 전송하지 않습니다. Client 컴포넌트는 꼭 필요한 곳에서만 사용합니다 — 블로그 태그 필터, 테마 토글, 모바일 내비게이션, 클립보드 복사, D2 다이어그램 전환기 — 각각 서버 렌더링된 부모를 감싸는 얇은 인터랙티브 래퍼입니다.

## 콘텐츠 시스템

콘텐츠는 코드와 함께 저장되는 파일 기반 MDX입니다. 블로그 포스트, 프로젝트 페이지, About 페이지 각각 로케일별 MDX 파일을 가집니다. 프로젝트는 세 탭 구조를 지원합니다: Overview(필수), Design(선택), Engineering(선택).

주목할 만한 설계 결정 두 가지:

**로케일 폴백.** 요청된 로케일이 없으면 `en.mdx`로 폴백합니다. 영어가 항상 기준선이고, 한국어 번역은 라우트를 깨뜨리지 않으면서 점진적으로 추가할 수 있습니다.

**Zod 프론트매터 검증.** 모든 프론트매터는 Zod 스키마로 파싱됩니다. 필수 필드가 없으면 에러를 throw합니다. 선택 필드는 `.catch(undefined)`를 사용하여 유효하지 않은 데이터를 조용히 드롭합니다. 잘못된 선택 필드 하나가 사이트 전체를 멈추는 일은 없습니다.

**스크립트별 읽기 시간.** 영어는 단어 수(200 WPM), 한국어는 글자 수(CJK 분당 300자)를 사용합니다. 콘텐츠가 아닌 로케일 기반 감지 — 단순하고 예측 가능합니다.

## i18n과 SSG 제약

다국어 처리는 `next-intl`로 두 로케일(`en`, `ko`)을 지원합니다. UI 문자열은 JSON 번역 파일에, 콘텐츠 번역은 별도 MDX 파일에 위치합니다.

핵심 SSG 함정: 서버 컴포넌트는 `getLocale()`가 아니라 페이지의 `params`에서 `setRequestLocale(locale)`을 호출해야 합니다. `getLocale()`는 정적 생성 중 존재하지 않는 요청 시점 컨텍스트에 의존하여 빌드를 조용히 깨뜨립니다.

번역이 필요한 async 서버 컴포넌트는 async 외부 컴포넌트(params 처리, 데이터 fetch)와 sync 내부 컴포넌트(`useTranslations()` 호출)로 분리하는 우회 패턴을 사용합니다. 번역 설정 역시 동적 `import()` 대신 명시적 정적 import를 사용하여 SSG 호환성을 확보합니다.

## D2 다이어그램 파이프라인

아키텍처 다이어그램은 선언적 다이어그래밍 언어인 D2로 작성합니다. 빌드 시 각 `.d2` 파일을 두 번 컴파일 — 라이트 테마 한 번, 다크 테마 한 번 — 쌍을 이루는 SVG를 생성합니다.

`D2Diagram` 컴포넌트는 hydration 안전을 위해 `useSyncExternalStore`를 사용합니다: 서버에서는 라이트 테마를 렌더링하고, 클라이언트 마운트 시 사용자 테마로 전환합니다. Hydration 불일치를 방지하면서 테마 인식 다이어그램을 유지합니다.

```tsx
<img src="/diagrams/arch-light.svg" className="dark:hidden" />
<img src="/diagrams/arch-dark.svg" className="hidden dark:block" />
```

## SEO

모든 페이지가 `generateMetadata()`를 사용하여 로케일별 title, description, canonical URL, hreflang 대체 링크, Open Graph 태그를 생성합니다. 중앙화된 `lib/seo.ts` 모듈이 페이지 메타데이터, canonical URL, 대체 링크, 구조화 데이터(website, person, article, project, breadcrumb, FAQ 타입의 JSON-LD) 빌더를 제공합니다.

JSON-LD 출력은 script 태그에서의 XSS를 방지하기 위해 꺾쇠괄호와 앰퍼샌드를 이스케이프합니다. 추가 SEO 인프라로 hreflang이 포함된 자동 생성 XML 사이트맵, 영어 블로그 포스트 기반 RSS 피드, robots.txt, LLM 탐색을 위한 llms.txt 파일이 있습니다.

## 보안

보안 헤더(CSP, HSTS, X-Frame-Options)는 `next.config.ts`에서 설정합니다. CSP `frame-src`는 YouTube와 Giscus만 허용합니다. "백엔드 없음" 제약은 공격 면이 설계상 최소화된다는 의미입니다 — API 엔드포인트 없음, 폼 처리 없음, 사용자 입력 처리 없음, 데이터베이스 연결 없음.
