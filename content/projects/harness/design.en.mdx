## Core Observation

Using oh-my-claudecode across multiple projects revealed a pattern. Agents and skills get rewritten every time. But the orchestration — delegation enforcement, progress tracking, completion verification, error recovery — stays identical. Out of 200+ files, only the orchestration core was reusable. Harness extracts that core.

Its most important job: stopping AI agents from doing everything themselves. Agents don't delegate by default. If a tool is available, they use it. That's efficient for small tasks, but at scale, it collides with the limits of a single context window.

## Why It's Built This Way

**Delegation is enforced by hooks, not prompts.** A system prompt saying "please delegate" gets ignored. Agents use whatever tools are available — prompts are suggestions, not constraints. So Harness intercepts Write, Edit, and NotebookEdit at the PreToolUse hook. The default mode is `warn` (injects a reminder message); setting it to `strict` blocks execution entirely. Source files (`.ts`, `.py`, `.rs`) and config files (`.harness/`, `CLAUDE.md`) are distinguished — source gets stronger warnings. Bash is also monitored: read-only commands (`ls`, `git diff`, `bun test`) pass through; modifying commands get a delegation reminder.

**Modes start with natural language.** Type "autopilot refactor the auth module" and pipeline mode begins. Code blocks are skipped, matching is case-insensitive, Korean works ("계속", "파이프라인", "중단"). Occasional false positives happen, but that's better than typing explicit commands every time.

**State lives on disk.** Mode, phase, and data are written to `.harness/state.json`. Session crashes, context compaction — the next session auto-restores. Working memory (`<remember>` tags) persists in `.harness/memory.json` and comes back too. A single JSON file is enough — one process reads and writes, no concurrency to worry about.

**Sub-agents lie.** When a sub-agent reports completion, the orchestrator receives a message: "Do not trust this claim. Run the tests yourself, read the code, check the build." A git diff summary is included. Not trusting self-reports is the orchestrator's reason for existing.
