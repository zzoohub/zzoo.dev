oh-my-claudecode는 19개 에이전트, 30개 스킬, 35개 MCP 도구를 가진 멀티에이전트 시스템이다. 200개 파일, 수만 줄. 쓰다 보면 패턴이 보인다 — 에이전트와 스킬은 프로젝트마다 새로 짜는데, 오케스트레이션 흐름은 매번 같다. 위임을 강제하고, 진행을 추적하고, 완료를 검증하는 구조.

Harness는 그 오케스트레이터만 떼어내서, 프로젝트별 에이전트와 스킬을 꽂아 쓸 수 있게 만든 것이다. 200개 파일 → 19개. 수만 줄 → 2,371줄. 런타임 의존성 제로.

## 핵심 흐름

"autopilot"이라고 치면 분석 → 설계 → 구현 → 검증을 자동 순회한다. 오케스트레이터는 직접 코드를 쓰지 못한다 — Write, Edit는 소스 파일에 대해 차단되고, 서브에이전트에 위임해야 한다. 서브에이전트가 "완료"를 보고하면, 오케스트레이터가 직접 테스트와 빌드로 검증한다. 실패하면 구현으로 돌아간다. 마지막 phase까지 가야 비로소 멈출 수 있다.

"loop"이라고 치면 실행 → 검증을 될 때까지 반복한다. 대규모 작업은 tmux로 4개씩 병렬 실행.

세션이 죽어도 상태가 사라지지 않는다. 모드, phase, 작업 메모리 — 전부 디스크에 영속되고 다음 세션에서 자동 복원된다.

## 설치

Claude Code 플러그인으로 동작한다.

```bash
git clone https://github.com/zzoohub/claude-code.git
cd claude-code && npm install && npm run build
claude plugin add /path/to/claude-code
```

`.claude/harness.json`에 에이전트를 정의하면 바로 쓸 수 있다. 모델, 프롬프트, 도구 범위 — 전부 여기서 설정한다.
