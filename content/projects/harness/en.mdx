Claude Code reads, writes, and tests code on its own. But push it on complex tasks and the limits show — context overflows, verification gets skipped, the thread loses track. An orchestrator harness is the layer that fixes this. It hooks into Claude Code's lifecycle to split work across sub-agents, track progress, and verify completion.

oh-my-claudecode is one of the most widely used harnesses in the Claude ecosystem. 19 agents, 30 skills, 35 MCP tools — 200+ files, tens of thousands of lines. Use it across projects and a pattern emerges. Agents and skills get rewritten every time. But the orchestration flow — force delegation, track progress, verify completion — stays identical.

Claude Harness extracts that orchestrator into a lightweight, standalone harness. Plug in your own agents and skills per project. 19 files, 2,371 lines. Zero runtime dependencies.

## Core Flow

Type "autopilot" and it cycles through understand, plan, execute, verify. The orchestrator can't write code directly — Write and Edit are blocked on source files, forcing delegation to sub-agents. When a sub-agent reports "done," the orchestrator verifies with tests and builds. Failure loops back to execute. Only the last phase allows a clean exit.

Type "loop" for execute then verify on repeat until it passes. Large tasks parallelize via tmux — batches of four, automatic polling, retry on failure.

State survives session crashes. Mode, phase, working memory — all persisted to disk and auto-restored on the next session.

## Install

Works as a Claude Code plugin. Install directly from GitHub:

```bash
/plugin marketplace add zzoohub/claude-plugins
/plugin install claude-harness@zzoohub-plugins
```

Define your agents in `.claude/harness.json` — model, prompt, tool scope. That's all the setup there is.
