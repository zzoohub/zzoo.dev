Give Claude Code a complex task, and it handles everything. Explores the codebase, writes code, runs tests, fixes errors — all in one session. And that's the problem.

When one agent does everything, context overflows. Exploration pushes implementation details out of the window. Implementation skips verification. Error recovery loses track of the original goal. The agent says "done" — but tests never ran and half the work is missing.

Harness takes the code-writing tools away from the orchestrator. Write, Edit, NotebookEdit — all blocked on source files. The only thing it can do is delegate to sub-agents.

## Hooks and State Machine

Two axes.

**Hooks** intercept five lifecycle events in Claude Code: session start, prompt submit, pre/post tool use, and stop. At each point, the harness injects context, blocks tools, or forces verification.

**State machine** records mode and phase to disk. "Pipeline mode, execute phase, one retry spent" survives session crashes, context compaction, even restarts. The next session picks up exactly where the last one stopped.

## Type "autopilot"

Cycles through understand, plan, execute, verify. When a phase ends and Claude tries to stop, the stop guard blocks it and advances to the next phase. Verification failure loops back to execute — up to three retries. Only the last phase allows a clean exit.

Type "loop" for execute then verify on repeat until it passes. Large tasks parallelize via tmux — batches of four, automatic polling, retry on failure.

Extracted from oh-my-claudecode. 200+ files down to 19. Tens of thousands of lines down to 2,371. Zero runtime dependencies.

## Install

Works as a Claude Code plugin.

```bash
claude plugin add /path/to/claude-harness
```

Define your agents in `.claude/harness.json` — model, prompt, tool scope. That's all the setup there is.
