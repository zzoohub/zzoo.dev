## Arquitectura

**Móvil (React Native / Expo)**
- Expo Router para navegación basada en archivos
- Zustand para estado del cliente, TanStack Query para estado del servidor
- MMKV para persistencia local rápida
- Bilingüe (EN/KO) via i18next

**API (Rust / Axum)**
- SQLx con queries verificadas en compile-time contra Neon PostgreSQL
- Autenticación JWT + JWKS (Google & Apple OAuth)
- Uploads con URL presignadas a Cloudflare R2
- Spec OpenAPI con Swagger UI

**Infraestructura (Pulumi)**
- GCP Cloud Run con autoscaling (0–3 instancias)
- Neon managed PostgreSQL
- Cloudflare R2 para almacenamiento de fotos
- GCP Secret Manager para credenciales
- GitHub Actions CI/CD para builds y deploys automatizados

## Tech Stack en Detalle

| Capa | Tecnología | Por Qué |
|------|-----------|---------|
| Móvil | React Native + Expo 55 | Cross-platform con performance nativo, actualizaciones OTA via EAS |
| Estado | Zustand + TanStack Query | Separación limpia de estado del cliente vs. servidor |
| API | Rust + Axum 0.8 | Type safety, bajo consumo de memoria, cold starts rápidos en Cloud Run |
| Base de Datos | PostgreSQL 18 (Neon) | Postgres serverless con branching para dev/staging |
| Storage | Cloudflare R2 | Compatible con S3, cero cargos por egress para almacenamiento de fotos |
| IaC | Pulumi (TypeScript) | Gestión declarativa de infra para GCP, Cloudflare y Neon |
| Monitoreo | Sentry + PostHog | Error tracking + product analytics |
