## Problema

La mayoría de las apps de seguimiento de comidas exigen input manual tedioso — buscar en bases de datos, pesar porciones, escribir descripciones. La fricción es tan alta que los usuarios abandonan el registro en días. Quería un rastreador de comidas que se sintiera tan natural como tomar una foto: snap, tag, listo.

## Solución

Mealio es una app de seguimiento de comidas basada en fotos diseñada alrededor de un solo principio: **registrar una comida en menos de 1 segundo**. La cámara es la interfaz principal. Los usuarios toman una foto, agregan detalles nutricionales opcionales, y construyen un diario visual de comida con el tiempo.

### Decisiones Clave de Diseño

- **UX centrada en la cámara** — El flujo de captura es la experiencia central. Foto primero, detalles opcionales. Esto reduce drásticamente la barrera para un registro consistente.
- **Arquitectura dual-mode** — El modo invitado (almacenamiento local, máximo 10 entradas) permite a los usuarios probar la app al instante. El modo autenticado desbloquea sincronización en la nube, entradas ilimitadas y estadísticas.
- **Feature-Sliced Design** — El codebase móvil sigue límites estrictos de capas (`app → widgets → features → entities → shared`) para un crecimiento mantenible.
- **API en Rust** — Elegí Axum sobre Node.js por type safety, performance y eficiencia de memoria en el autoscaling de Cloud Run (0–3 instancias).

## Resultados

- **Flujo de captura en 1 segundo** — Desde tocar la cámara hasta la entrada guardada, consistentemente en menos de 1 segundo
- **Cross-platform** — Un solo codebase sirviendo tanto iOS (App Store) como Android (Play Store)
- **Autoscaling de cero a tres** — Cloud Run escala a cero cuando está inactivo, manteniendo costos mínimos
- **9 migraciones de base de datos** — Evolución estable del schema sin downtime
- **4 pipelines de CI/CD** — Deploy automatizado de API, CI móvil, builds de EAS y actualizaciones OTA
