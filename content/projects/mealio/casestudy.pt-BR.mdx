## Problema

A maioria dos apps de rastreamento de refeições exige input manual tedioso — pesquisar bancos de dados, pesar porções, digitar descrições. O atrito é tão alto que os usuários abandonam o registro em poucos dias. Eu queria um rastreador de refeições que fosse tão natural quanto tirar uma foto: snap, tag, pronto.

## Solução

Mealio é um app de rastreamento de refeições com foco em foto, projetado em torno de um único princípio: **registrar uma refeição em menos de 1 segundo**. A câmera é a interface principal. Usuários tiram uma foto, adicionam detalhes nutricionais opcionais e constroem um diário visual de alimentação ao longo do tempo.

### Decisões de Design Chave

- **UX centrada na câmera** — O fluxo de captura é a experiência central. Foto primeiro, detalhes opcionais. Isso diminui drasticamente a barreira para registros consistentes.
- **Arquitetura dual-mode** — Modo visitante (armazenamento local, máximo 10 registros) permite que usuários experimentem o app instantaneamente. Modo autenticado libera sync na nuvem, registros ilimitados e estatísticas.
- **Feature-Sliced Design** — A codebase mobile segue limites estritos de camada (`app → widgets → features → entities → shared`) para crescimento sustentável.
- **API Rust** — Escolhi Axum em vez de Node.js por type safety, performance e eficiência de memória no autoscaling do Cloud Run (0-3 instâncias).

## Resultados

- **Fluxo de captura em 1 segundo** — Do toque na câmera até o registro salvo, consistentemente abaixo de 1 segundo
- **Cross-platform** — Codebase única servindo tanto iOS (App Store) quanto Android (Play Store)
- **Autoscaling de zero a três** — Cloud Run escala para zero quando ocioso, mantendo custos mínimos
- **9 migrações de banco de dados** — Evolução estável de schema sem downtime
- **4 pipelines CI/CD** — Deploy automatizado de API, CI mobile, builds EAS e atualizações OTA
