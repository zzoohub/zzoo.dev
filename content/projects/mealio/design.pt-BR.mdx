## Arquitetura

**Mobile (React Native / Expo)**
- Expo Router para navegação baseada em arquivos
- Zustand para estado do cliente, TanStack Query para estado do servidor
- MMKV para persistência local rápida
- Bilíngue (EN/KO) via i18next

**API (Rust / Axum)**
- SQLx com queries verificadas em tempo de compilação contra Neon PostgreSQL
- Autenticação JWT + JWKS (Google & Apple OAuth)
- Uploads via URL pré-assinada para Cloudflare R2
- Spec OpenAPI com Swagger UI

**Infraestrutura (Pulumi)**
- GCP Cloud Run com autoscaling (0-3 instâncias)
- Neon managed PostgreSQL
- Cloudflare R2 para armazenamento de fotos
- GCP Secret Manager para credenciais
- GitHub Actions CI/CD para builds e deploys automatizados

## Tech Stack em Profundidade

| Camada | Tecnologia | Por quê |
|--------|-----------|---------|
| Mobile | React Native + Expo 55 | Cross-platform com performance nativa, atualizações OTA via EAS |
| Estado | Zustand + TanStack Query | Separação limpa de estado do cliente vs. servidor |
| API | Rust + Axum 0.8 | Type safety, baixo consumo de memória, cold starts rápidos no Cloud Run |
| Banco de Dados | PostgreSQL 18 (Neon) | Postgres serverless com branching para dev/staging |
| Storage | Cloudflare R2 | Compatível com S3, zero taxas de egress para armazenamento de fotos |
| IaC | Pulumi (TypeScript) | Gestão declarativa de infra para GCP, Cloudflare e Neon |
| Monitoramento | Sentry + PostHog | Rastreamento de erros + analytics de produto |
