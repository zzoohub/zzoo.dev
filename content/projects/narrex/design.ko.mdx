## 핵심 인사이트

작가는 장면과 플롯 흐름 단위로 사고합니다 — 문장이 아니라. 기존 AI 글쓰기 도구는 이걸 무시합니다. 문장 단위 자동완성은 구조 작업을 전부 작가에게 떠넘기고, 완전 자동 생성은 창작 주도권을 빼앗습니다. 둘 다 진짜 병목을 건드리지 않습니다: 수십 개의 연결된 장면을 일관된 다회차 서사로 발전시키는 작업.

Narrex는 장면을 AI 생성의 단위로, 비주얼 구조를 프롬프트로 만듭니다.

## 누구를 위한 제품인가

**지연 — 지망생 작가**

회귀 판타지를 몇 달째 상상 중인 대학생. 캐릭터 이름도 정했고 첫 장면도 그려집니다. 하지만 아이디어에서 40화 완성 원고까지의 거리에는 한 번도 넘어본 적 없는 벽이 있습니다. ChatGPT를 써봤지만 몇 챕터 후 일관성이 무너졌습니다. 부족한 건 창의력이 아닙니다. 생산 시스템이 없는 겁니다.

*"이야기 아이디어를 진짜 읽을 수 있는 소설로 만들고 싶어요 — 산문 기술을 마스터하지 않아도."*

**민호 — 연재 중인 작가**

노벨피아에 8화를 올렸습니다. 독자들이 전제는 좋아했지만 일관성 문제를 지적했습니다 — 캐릭터 눈 색이 바뀌고, 복선이 회수되지 않았습니다. 퇴근 후 2~3시간 써서 주 1화가 한계. 이야기는 알고 있지만 독자가 기대하는 속도에서 품질을 유지할 수 없습니다.

*"이전에 무슨 일이 있었는지 도구가 기억하고, 기존 이야기와 일관된 초안을 생성해줬으면 해요."*

## 코어 루프

모든 설계 결정은 하나의 유저 저니에서 출발합니다:

1. **스토리 아이디어 붙여넣기** — 어떤 출발점이든, 대략적인 한 문단이면 충분
2. **자동 구조 생성** — 설정, 장면이 배치된 타임라인, 관계가 연결된 캐릭터 맵
3. **시각적으로 이야기 배치** — 장면 드래그, 병렬 스토리라인용 트랙 추가, 캐릭터 편집
4. **장면별 산문 생성** — AI가 전체 컨텍스트를 자동 조립해 장면 초안 작성
5. **연출과 수정** — 자유롭게 편집하거나 "긴장감을 높여줘" 같은 연출 지시

첫 가치까지 3클릭: 프로젝트 생성, 아이디어 입력, "내 이야기 구성하기" 클릭. 소설 쓰기의 가장 어려운 부분 — "아이디어가 있다"에서 "계획이 있다"로 가는 것 — 이 첫 세션에서 일어납니다.

## 핵심 제품 결정

### 장면 단위 생성

근본적인 질문: AI가 어떤 단위로 생성하는가?

- **문장 단위** (Sudowrite, TypeTak): 구조 작업은 전부 작가 몫. AI는 문장만 이어줌.
- **완전 자동 생성** (ChatGPT, 독립형 생성기): 작가 통제 없음. 몇 번의 대화 후 일관성 소멸.
- **장면 단위** (Narrex): 작가가 구조를 시각적으로 구축. AI가 전체 서사 컨텍스트 안에서 산문을 생성.

장면 단위가 스윗 스팟입니다 — 일관된 출력을 위한 충분한 컨텍스트, 작가 통제를 위한 충분한 세분화.

### NLE 타임라인 멘탈 모델

워크스페이스는 영상 편집기를 모델로 합니다. 멀티트랙 클립(장면)이 있는 타임라인, 프리뷰 윈도우(에디터), 미디어 라이브러리(캐릭터 맵). 이 메타포를 선택한 이유:

- 작가는 자연스럽게 병렬 스토리라인을 생각함 — 멀티트랙이 이를 직접 지원
- 장면이 위치뿐 아니라 범위(duration)를 가짐 — 트랙 간 동시성 감지 가능
- NLE 메타포는 Premiere, CapCut, GarageBand를 써본 디지털 네이티브에게 직관적

### 사건-에피소드 분리

장면(이야기에서 무슨 일이 일어나는지)과 에피소드(독자에게 어떻게 전달되는지)는 독립적인 레이어입니다. 핵심적인 제품 결정:

- 여러 작은 사건이 하나의 에피소드로 합쳐질 수 있음
- 하나의 큰 사건이 여러 에피소드에 걸칠 수 있음
- 타임라인의 에피소드 구분선을 드래그하면 챕터가 재분할되고, AI 프롬프트가 자동 재구성

이 구분을 만드는 도구는 다른 곳에 없습니다. 모든 경쟁자가 챕터를 기본 단위로 취급합니다.

### "처음부터 쓰지 말고, 수정하세요"

AI가 초안을 생성합니다. 작가가 연출하고 다듬습니다. 이 프레이밍은 의도적입니다 — 사용자의 역할을 "못 쓰는 작가"에서 "결과물을 다듬는 연출가"로 전환합니다. 연출 기반 편집("긴장감을 높여줘", "대화를 늘려줘")이 기술 장벽을 낮추면서도 창작 소유권을 유지합니다.

## UX 설계

### 워크스페이스: 단일 페이지, 패널 기반

깊은 네비게이션 없음. 사용자는 페이지가 아니라 패널을 전환합니다. 모든 핵심 액션이 워크스페이스에서 최대 2클릭.

```
+---------------------------------------------------------------+
| 헤더 · 프로젝트 제목              [설정 v]  [테마]  저장됨      |
+----------+------------------------------------+---------------+
| 캐릭터   | 에디터                             | 장면 상세    |
| 맵       | (산문 프리뷰 + 편집)               | (메타데이터)  |
+----------+--------+---------------------------+---------------+
| 타임라인 (멀티트랙, 가로 스크롤, 드래그 앤 드롭)              |
| 트랙 1: [장면]--[장면]--[장면]--[장면]--[+]                   |
| 트랙 2:         [장면]--[장면]--[+]                            |
+---------------------------------------------------------------+
```

### 프로그레시브 디스클로저

비기술적 취미 작가가 주요 사용자입니다. 인터페이스가 압도하면 안 됩니다.

| 단계 | 보이는 것 | 숨겨진 것 | 공개 시점 |
|------|----------|----------|----------|
| 첫 로드 | 싱글트랙 타임라인, 설정 바, 캐릭터 맵 | 멀티트랙, 분기/병합, 컨텍스트 메뉴 | 타임라인 상호작용 시 |
| 첫 장면 선택 | 장면 상세 패널 자동 오픈 | 연출 기반 편집, 단축키 | 첫 초안 생성 후 |
| 첫 초안 생성 | 텍스트 편집 | "AI로 수정" | 텍스트 선택 시 |

### 장면 비주얼 상태

색상만이 아닌 — 접근성을 위해 채움 패턴 + 아이콘 + 레이블 병용:

- **미작성** — 빈 클립, 아이콘 없음
- **AI 초안** — 반쯤 채워진 클립, 펜 아이콘
- **수정 완료** — 꽉 찬 클립, 체크마크
- **재확인 필요** — 경고 테두리, 삼각형 아이콘 (마지막 초안 이후 설정 변경)

### 자동 저장, 저장 버튼 없음

모든 편집이 1초 디바운스로 자동 저장. 상태 표시: "저장 중..." → "모든 변경사항 저장됨." 단일 사용자 제품이므로 last-write-wins로 충분합니다.

## 경쟁 포지셔닝

8개 도구 검토 (해외 4개, 한국 4개). 비주얼 타임라인 + 장면 단위 AI + 한국 웹소설 특화를 결합한 도구는 없습니다.

| 간극 | 해외 도구 | 한국 도구 | Narrex |
|------|---------|---------|--------|
| 비주얼 타임라인 | 제공 없음 | 제공 없음 | 멀티트랙 NLE |
| AI 생성 단위 | 문장(Sudowrite) 또는 없음(Novelcrafter) | 완전 자동(독립형) 또는 문장(TypeTak) | 전체 컨텍스트 반영 장면 단위 |
| 한국 웹소설 장르 | 영어 전용 | 낮은 출력 품질 | 회귀물, 로맨스, 무협 관습에 최적화 |
| 작가 통제 | 다양 | 최소 | 구조가 곧 프롬프트 — 완전한 통제 |

## 비즈니스 모델

| 플랜 | 가격 | 포함 내용 |
|------|------|----------|
| Free | $0 | 무제한 구조 편집. 월 50회 AI 생성. |
| Basic | ~$12/월 | 월 100회 생성. 초안 2개 변형. 톤 슬라이더. |
| Pro | ~$25/월 | 무제한 생성. 3개 변형. 고급 모델. 리비전 도구. |

장면당 비용: ~$0.03~0.06 (프롬프트 캐싱 적용). 40화 소설 완성 비용: ~$3.60~9.60. 생성 한도 도달 시 무료→유료 전환 유도.

## 페이즈 전략

**Phase 1 (현재):** 코어 루프 검증 — 아이디어 → 구조 → 생성 → 편집. 단 하나의 질문: 사용자가 AI 초안을 계속 수정하는가, 아니면 버리는가?

**Phase 2:** 에피소드 편성 레이어, 초안 변형, 톤 슬라이더, 내보내기.

**Phase 3:** 월드 맵, 시간별 관계 추적, 리비전 도구, AI 서프라이즈 모드.
