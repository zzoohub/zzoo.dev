## 아키텍처

**모듈러 모놀리스 + 헥사고날 (Rust / Axum)**

솔로 개발자가 긴밀하게 결합된 도메인을 구축합니다 — 프로젝트, 장면, 캐릭터, 생성이 모두 데이터를 공유합니다. 모놀리스로 배포 오버헤드를 없애고, 도메인별 모듈은 Rust trait을 포트로 사용해 깔끔하게 분리합니다. 필요하면 서비스로 추출 가능. 서브밀리초 응답, 10~30MB 메모리, 컴파일 타임 안전성.

`llm` 크레이트는 별도 워크스페이스 크레이트입니다 — 컴파일러가 API 내부를 물리적으로 임포트하지 못하게 강제합니다. "LLM과 어떻게 대화할지"와 "무엇을 물어볼지"의 경계를 강제하는 구조입니다.

```
apps/api/
├── crates/llm/              # LLM 프로바이더 추상화 (독립)
├── src/domain/               # 비즈니스 로직 (인프라 의존성 없음)
│   ├── project/              # 프로젝트 CRUD, 스토리 설정
│   ├── timeline/             # 트랙, 장면, 분기/병합
│   ├── character/            # 캐릭터, 관계
│   └── ai/                   # 프롬프트 조립, 생성
├── src/inbound/http/         # HTTP 어댑터 (Axum 핸들러)
└── src/outbound/             # 인프라 어댑터 (SQLx, R2, JWT)
```

**프론트엔드: TanStack Start + SolidJS**

타임라인 UI 때문에 React 대신 SolidJS를 선택했습니다. 파인그레인드 반응성으로 드래그 앤 드롭 시 각 장면 노드가 독립적으로 업데이트됩니다 — 수백 개 엘리먼트에 걸친 가상 DOM 디핑이 필요 없습니다. Nitro 어댑터를 통해 Cloudflare Workers에 배포.

## 데이터베이스 설계

**Phase 1에 11개 테이블.** 핵심 설계 결정:

- **NLE 포지셔닝** — `scene.start_position`과 `scene.duration`을 `DOUBLE PRECISION`으로. 장면은 리스트 항목이 아니라 범위를 가진 클립입니다. 초기 간격 1024.0으로 리밸런싱 없이 ~50회 중간점 삽입 가능. 트랙 간 동시성은 범위 겹침으로 판단.

- **초안 버전 관리** — 별도 `draft` 테이블에 `(scene_id, version)`. 전체 편집 히스토리를 보존하고, 빈번히 읽는 장면 메타데이터에서 큰 텍스트 블롭을 분리합니다. 각 초안은 소스(`ai_generation`, `ai_edit`, `manual`), 모델, 토큰 수를 기록합니다.

- **컨텍스트 압축** — `scene_summary` 테이블(장면과 1:1)에 AI 압축 요약(~200~300 토큰) 저장. 생성 프롬프트에서 전체 장면 텍스트 대신 이전 요약을 사용하여 15개 이상의 장면에서도 컨텍스트를 관리 가능하게 유지합니다.

- **생성 비용 추적** — `generation_log`의 프로젝트/장면 FK에 `ON DELETE SET NULL` 적용. 엔티티 삭제 후에도 비용 데이터가 과금 분석용으로 보존됩니다.

## 데이터 플로우

네 가지 주요 플로우, 모두 `LlmGateway`를 통한 SSE 스트리밍:

1. **아이디어에서 프로젝트로** — 사용자가 스토리 아이디어를 붙여넣으면 LLM이 구조화된 JSON(프로젝트 설정, 트랙, 장면, 캐릭터)을 반환. 단일 트랜잭션으로 Postgres에 기록. 막연한 아이디어가 몇 초 만에 비주얼 타임라인으로 변환.

2. **장면 초안 생성** — 시스템이 6개 소스에서 컨텍스트를 조립: 스토리 설정, 장면 상세, 캐릭터 카드, 선행 장면 압축 요약, 병렬 트랙의 동시 사건, 다음 장면 프리뷰(총 ~2,500~5,000 토큰). 완료 시 초안이 버전으로 기록. 기존 콘텐츠가 없으면 자동 적용, 있으면 사용자 확인 후 교체.

3. **디렉션 기반 편집** — 사용자가 문단을 선택하고 "더 긴장감 있게" 같은 디렉션을 입력. 생성과 동일한 컨텍스트 조립에 선택된 텍스트를 추가. 새 초안 버전이 `ai_edit`로 기록.

4. **에디터 자동 저장** — 편집은 1~2초 디바운스 후 직접 PATCH. 첫 수동 수정 시 장면 상태가 `ai_draft`에서 `edited`로 전환.

## AI / LLM 아키텍처

**`crates/llm`을 통한 프로바이더 추상화:**

```rust
#[async_trait]
pub trait LlmProvider: Send + Sync {
    async fn generate(&self, req: GenerateRequest) -> Result<GenerateResponse>;
    async fn generate_stream(&self, req: GenerateRequest)
        -> Result<Pin<Box<dyn Stream<Item = Result<String>> + Send>>>;
}
```

`LlmGateway`가 페일오버 처리: Gemini Flash-Lite 우선, 실패 시(타임아웃, 5xx, 429) CF Workers AI로 전환.

**장면당 컨텍스트 어셈블리 (~2,500~5,000 토큰):**

| 소스 | 데이터 | 토큰 |
|------|--------|------|
| 시스템 인스트럭션 | 생성 규칙, 출력 포맷 | ~500 |
| 글로벌 설정 | 장르, 테마, 시대, 시점, 분위기 | ~100 |
| 장면 상세 | 제목, 줄거리, 장소, 분위기 | ~200 |
| 캐릭터 + 관계 | 관련 캐릭터 카드 | ~350~700 |
| 이전 장면 요약 | 압축된 선행 장면 요약 | ~1,000~3,000 |
| 동시 사건 | 다른 트랙의 겹치는 장면 | ~200~400 |
| 다음 장면 프리뷰 | 제목 + 줄거리 | ~100 |

동일한 데이터가 주어지면 어셈블리는 결정적입니다 — 프롬프트 구성에 랜덤성이 없습니다.

**컨텍스트 압축:** 각 장면 초안 완료 후 시스템이 ~200~300 토큰 요약을 생성합니다. 핵심 플롯 이벤트, 캐릭터 상태, 복선을 보존합니다. Postgres(영구)와 Redis(빠른 읽기)에 저장. 15개 장면, 250토큰 요약 기준 약 3,750 토큰 — 컨텍스트 한도 내에서 충분.

**스트리밍:** Axum 네이티브 `Sse` 타입 + `async-stream`으로 SSE. 세 가지 엔드포인트: 장면 생성, 디렉션 기반 편집, 프로젝트 자동 구조화(3단계: 세계관, 캐릭터, 타임라인). 프론트엔드는 SolidJS에서 커스텀 `fetch` + `ReadableStream` 파서로 소비 — SolidJS 어댑터가 없어 Vercel AI SDK 의존성 없음.

## 핵심 결정

**Rust/Axum over FastAPI** — Python 전용 라이브러리가 필요 없음. LLM 호출은 `reqwest`로 HTTP 통신. Rust는 서브밀리초 레이턴시, 10~30MB 메모리, 컴파일 타임 안전성을 최저 클라우드 비용으로 제공.

**모놀리스 over 마이크로서비스** — 개발자 1명, 배포 단위 1개. 생성이 모든 다른 모듈에서 읽음. 하나의 배포 = 하나의 CI 파이프라인, 하나의 컨테이너, 하나의 로그 스트림. 팀이 성장하거나 생성에 GPU 컴퓨트가 필요하면 재고.

**Neon over Supabase** — 제로 스케일 과금(출시 시 무료), 브랜치 기반 개발 워크플로우, 향후 임베딩용 pgvector. Cloud Run과 버지니아에 공동 배치하여 낮은 크로스 클라우드 레이턴시 확보.

**`llm`을 별도 워크스페이스 크레이트로** — 컴파일러가 경계를 강제. 목 프로바이더로 독립 테스트 가능. CLI 도구나 워커에서 재사용 가능. 런타임 비용 제로 — 라이브러리이지 서비스가 아님.

**SSE over WebSocket** — 토큰 전달에 필요한 건 서버에서 클라이언트로의 단방향 스트리밍뿐. SSE는 상태 비저장, HTTP 기반, Cloud Run에서 간단하게 처리. WebSocket은 이점 없이 연결 관리 복잡성만 추가. 실시간 협업이 추가되면 재고.

**Google OAuth만 (Phase 1)** — 가장 단순한 구현. 카카오 추가는 양방향 문 — 또 하나의 OAuth 프로바이더일 뿐, 몇 주면 추가 가능. 획득 데이터에서 로그인 이탈이 보이면 재고.
