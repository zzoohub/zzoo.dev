## Arquitetura

Idea Fork usa uma arquitetura de caminho dividido: a web app Next.js le diretamente do Neon PostgreSQL para todas as consultas de usuario, enquanto o backend FastAPI roda apenas como pipeline de dados diario.

**Web App (Next.js no Vercel)**
- App Router com SSR/ISR, arquitetura Feature-Sliced Design (FSD)
- Acesso dual de dados: `@neondatabase/serverless` para leituras SQL diretas, Server Actions para escritas (ratings)
- next-intl para 6 locales (en, es, pt-BR, id, ja, ko)
- Animacoes GSAP, tema escuro/claro

**API + Pipeline (FastAPI no GCP Cloud Run)**
- Arquitetura hexagonal: padrao de portas/adaptadores de dominio com Protocol ABCs
- Pipeline diario de 5 estagios: Fetch, Tag, Score, Cluster, Synthesize
- Advisory lock previne execucoes sobrepostas do pipeline

**Banco de Dados (Neon PostgreSQL)**
- 10 tabelas: post, tag, post_tag, cluster, cluster_post, brief, brief_source, product, product_tag, rating
- Paginacao keyset baseada em cursor (cursores JSON codificados em base64)
- Indices parciais em `deleted_at IS NULL` para soft deletes

## Stack Tecnico

| Camada | Tecnologia | Razao |
|--------|-----------|-------|
| Web | Next.js 16 + React 19 | App Router SSR/ISR, streaming, Server Actions |
| Estilo | Tailwind CSS v4 | Utility-first, modo escuro com temas oklch |
| Animacao | GSAP + @gsap/react | Animacoes de scroll e entrada de alto desempenho |
| API | Python 3.12 + FastAPI | Ecossistema rico em ML/LLM (scikit-learn, google-genai) |
| Arquitetura | Hexagonal (portas/adaptadores) | Multiplos adaptadores externos intercambiaveis |
| LLM | Google Gemini (Flash Lite + Flash + Embeddings) | Tagueamento, clustering e sintese de briefs a baixo custo |
| Clustering | HDBSCAN (scikit-learn) | Deteccao automatica de numero de clusters |
| Banco de Dados | PostgreSQL 18 (Neon) | Preco scale-to-zero, driver serverless |
| IaC | Pulumi (TypeScript) | Gestao declarativa para GCP, Neon e Vercel |
| Analytics | PostHog | Analytics de produto com 9 eventos personalizados |
| Erros | Sentry | Monitoramento de erros full-stack |
