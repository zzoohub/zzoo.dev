## Problema

Indie hackers y fundadores en etapa temprana pasan mas de 5 horas por semana investigando manualmente oportunidades de producto en Reddit, Hacker News, Product Hunt y tiendas de apps. El proceso esta fragmentado: las senales de demanda (quejas de usuarios) estan en un lugar, los datos de oferta (productos existentes) en otro, y sintetizar la brecha requiere un esfuerzo manual significativo. La mayor parte de esta investigacion produce un spreadsheet desordenado y una corazonada vaga — no insights accionables.

## Solucion

Idea Fork automatiza todo el pipeline de investigacion demanda-oferta de principio a fin. Un pipeline de datos diario de 5 etapas recolecta quejas de usuarios de 44 subreddits, feeds RSS y resenas de app stores, rastrea productos trending de Product Hunt y app stores, y usa un pipeline de Gemini LLM para sintetizar la brecha en briefs de oportunidad de producto respaldados por fuentes.

### Arquitectura de Ruta Dividida

La web app lee directamente de Neon PostgreSQL via `@neondatabase/serverless`, evitando completamente el servidor API para operaciones de lectura. El backend FastAPI se ejecuta solo como pipeline programado — no esta en la ruta de lectura de produccion.

### Backend Hexagonal

El servicio FastAPI usa arquitectura hexagonal con Protocol ABCs como puertos de dominio. Cada adaptador externo es independientemente testeable y los data sources son intercambiables sin tocar la logica de negocio.

### IA con Evidencia

Cada brief generado por IA incluye `source_snapshots` — JSONB desnormalizado que vincula cada afirmacion con su post original. Los usuarios pueden verificar las afirmaciones sin joins adicionales en la base de datos.

## Resultados

- **Pipeline automatizado de 5 etapas** — Ejecucion diaria en 5+ fuentes de datos sin intervencion manual
- **350+ briefs generados** — Cada uno con atribucion de fuentes vinculando a posts originales
- **i18n en 6 idiomas** — Internacionalizacion completa para en, es, pt-BR, id, ja, ko
- **Autoscaling zero-to-three** — Cloud Run escala a cero cuando el pipeline no se ejecuta
- **Esquema PostgreSQL de 10 tablas** — Diseno normalizado soportando posts, productos, briefs, clusters, tags y ratings
