## アーキテクチャ

Idea Forkは分離パスアーキテクチャを使用しています：Next.jsウェブアプリはすべてのユーザー向けクエリをNeon PostgreSQLから直接読み取り、FastAPIバックエンドは日次データパイプラインとしてのみ実行されます。

**ウェブアプリ (Next.js on Vercel)**
- App Router SSR/ISR、Feature-Sliced Design (FSD) アーキテクチャ
- デュアルデータアクセス：`@neondatabase/serverless`で直接SQL読み取り、Server Actionsで書き込み（評価）
- next-intlで6ロケール (en, es, pt-BR, id, ja, ko)
- GSAPアニメーション、ダーク/ライトテーマ

**API + パイプライン (FastAPI on GCP Cloud Run)**
- ヘキサゴナルアーキテクチャ：Protocol ABCによるドメインポート/アダプターパターン
- 5段階日次パイプライン：Fetch、Tag、Score、Cluster、Synthesize
- Advisory lockでパイプラインの重複実行を防止

**データベース (Neon PostgreSQL)**
- 10テーブル：post, tag, post_tag, cluster, cluster_post, brief, brief_source, product, product_tag, rating
- カーソルベースのキーセットページネーション（base64エンコードJSONカーソル）
- `deleted_at IS NULL`の部分インデックスでソフト削除

## 技術スタック

| レイヤー | 技術 | 理由 |
|---------|------|------|
| ウェブ | Next.js 16 + React 19 | App Router SSR/ISR、ストリーミング、Server Actions |
| スタイリング | Tailwind CSS v4 | ユーティリティファースト、oklchテーマのダークモード |
| アニメーション | GSAP + @gsap/react | 高性能スクロール・エントランスアニメーション |
| API | Python 3.12 + FastAPI | 豊富なML/LLMエコシステム（scikit-learn、google-genai） |
| アーキテクチャ | ヘキサゴナル（ポート/アダプター） | 交換可能な複数の外部アダプター |
| LLM | Google Gemini (Flash Lite + Flash + Embeddings) | 低コストのタグ付け、クラスタリング、ブリーフ合成 |
| クラスタリング | HDBSCAN (scikit-learn) | クラスター数の自動検出 |
| データベース | PostgreSQL 18 (Neon) | スケールトゥゼロ価格、サーバーレスドライバー |
| IaC | Pulumi (TypeScript) | GCP、Neon、Vercelの宣言的管理 |
| 分析 | PostHog | 9カスタムイベントのプロダクト分析 |
| エラー追跡 | Sentry | フルスタックエラーモニタリング |
