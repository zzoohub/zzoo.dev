## 아키텍처

Idea Fork는 분리 경로 아키텍처를 사용합니다: Next.js 웹 앱은 모든 사용자 대면 쿼리를 Neon PostgreSQL에서 직접 읽고, FastAPI 백엔드는 일일 데이터 파이프라인으로만 실행됩니다. 이를 통해 읽기 경로 지연을 제거하고 유휴 시 API 서버 비용을 거의 0에 가깝게 유지합니다.

**웹 앱 (Next.js on Vercel)**
- App Router SSR/ISR, Feature-Sliced Design (FSD) 아키텍처
- 이중 데이터 액세스: `@neondatabase/serverless`로 직접 SQL 읽기, Server Actions로 쓰기 (평가)
- next-intl로 6개 로케일 (en, es, pt-BR, id, ja, ko)
- GSAP 애니메이션, 다크/라이트 테마

**API + 파이프라인 (FastAPI on GCP Cloud Run)**
- 헥사고날 아키텍처: Protocol ABC를 활용한 도메인 포트/어댑터 패턴
- 5단계 일일 파이프라인: Fetch, Tag, Score, Cluster, Synthesize
- Advisory lock으로 파이프라인 중복 실행 방지
- 인바운드 어댑터: 공개용과 내부용 라우팅 레이어 분리
- 아웃바운드 어댑터: PostgreSQL (asyncpg), Gemini LLM, Reddit, RSS, App Store, Play Store, Product Hunt, Google Trends

**데이터베이스 (Neon PostgreSQL)**
- 10개 테이블: post, tag, post_tag, cluster, cluster_post, brief, brief_source, product, product_tag, rating
- 커서 기반 키셋 페이지네이션 (base64 인코딩 JSON 커서)
- `deleted_at IS NULL` 부분 인덱스로 소프트 삭제
- `tsvector`를 활용한 전문 검색

## 데이터 파이프라인 상세

파이프라인은 GCP Cloud Scheduler에 의해 매일 트리거되며 Cloud Run에서 실행됩니다 (1 vCPU, 1Gi, 0-3 인스턴스).

**1단계: Fetch** — `asyncio.gather()`를 통한 동시 데이터 수집, 세마포어 제어 리뷰 페칭 (동시 3개). 소스: Reddit (~44개 서브레딧), RSS (HN 50+ 포인트, TechCrunch), App Store (iTunes API, 20개 키워드 카테고리), Play Store (스크레이퍼, 20개 키워드), Product Hunt (GraphQL, 실행당 30개 제품).

**2단계: Tag** — 20개 게시물 배치를 Gemini Flash Lite로 전송. `post_type` (10종), `sentiment` (4종), 2-3개 토픽 태그 할당. 기존 태그 재사용이 포함된 Few-shot 프롬프트. 실패한 배치는 재시도로 표시.

**3단계: Score** — 매칭된 게시물 태그를 기반으로 제품의 `trending_score`와 `complaint_count` 업데이트.

**4단계: Cluster** — 200개 태그된 게시물 배치를 Gemini Embedding API로 벡터화, HDBSCAN으로 클러스터링 (`min_cluster_size=3`, `min_samples=2`). 노이즈 게시물은 "기타"로 그룹화. Gemini Flash Lite가 클러스터 라벨 생성.

**5단계: Synthesize** — 각 클러스터에 대해: Google Trends 데이터와 관련 제품을 병렬로 가져온 후, Gemini Flash (temperature 0.9)가 제목, 요약, 문제 정의, 기회, 솔루션 방향, 수요 시그널, 경쟁 환경, 소스 스냅샷이 포함된 전체 브리프 작성.

## 보안

- 엄격한 `default-src 'self'` CSP 헤더
- 프로덕션 도메인만 허용하는 CORS 제한
- 속도 제한: 세션당 10개 평가/분, IP당 100개 요청/분
- PII 미수집 — httpOnly 쿠키를 통한 익명 세션 기반 평가
- 모든 시크릿은 GCP Secret Manager에 저장
- Pydantic 검증 + SQLAlchemy 파라미터화된 쿼리
- `server-only` 패키지로 클라이언트 측 DB 임포트 방지
