## Architecture

Idea Fork uses a split-path architecture: the Next.js web app reads directly from Neon PostgreSQL for all user-facing queries, while a FastAPI backend runs only as a daily data pipeline. This eliminates read-path latency and keeps the API server cost at near-zero when idle.

**Web App (Next.js on Vercel)**
- App Router with SSR/ISR, Feature-Sliced Design (FSD) architecture
- Dual data access: `@neondatabase/serverless` for direct SQL reads, Server Actions for writes (ratings)
- next-intl for 6 locales (en, es, pt-BR, id, ja, ko)
- GSAP animations, dark/light theme

**API + Pipeline (FastAPI on GCP Cloud Run)**
- Hexagonal architecture: domain ports/adapters pattern with Protocol ABCs
- 5-stage daily pipeline: Fetch, Tag, Score, Cluster, Synthesize
- Advisory lock prevents overlapping pipeline runs
- Inbound adapters: separate public-facing and internal routing layers
- Outbound adapters: PostgreSQL (asyncpg), Gemini LLM, Reddit, RSS, App Store, Play Store, Product Hunt, Google Trends

**Database (Neon PostgreSQL)**
- 10 tables: post, tag, post_tag, cluster, cluster_post, brief, brief_source, product, product_tag, rating
- Cursor-based keyset pagination (base64-encoded JSON cursors)
- Partial indexes on `deleted_at IS NULL` for soft deletes
- Full-text search via `tsvector` on posts

## Data Pipeline Details

The pipeline is triggered daily by GCP Cloud Scheduler and runs on Cloud Run (1 vCPU, 1Gi, 0-3 instances).

**Stage 1: Fetch** — Concurrent data collection via `asyncio.gather()` with semaphore-controlled review fetching (3 concurrent). Sources: Reddit (~44 subreddits), RSS (HN 50+ pts, TechCrunch), App Store (iTunes API, 20 keyword categories), Play Store (scraper, 20 keywords), Product Hunt (GraphQL, 30 products/run).

**Stage 2: Tag** — Batches of 20 posts sent to Gemini Flash Lite. Assigns `post_type` (10 types), `sentiment` (4 values), and 2-3 topic tags. Few-shot prompt with existing tag reuse. Failed batches marked for retry.

**Stage 3: Score** — Updates `trending_score` and `complaint_count` on products based on matched post tags.

**Stage 4: Cluster** — Batches of 200 tagged posts vectorized via Gemini Embedding API, then clustered via HDBSCAN (`min_cluster_size=3`, `min_samples=2`). Noise posts grouped as "Miscellaneous." Cluster labels generated by Gemini Flash Lite.

**Stage 5: Synthesize** — For each cluster: fetches Google Trends data and related products in parallel, then Gemini Flash (temperature 0.9) writes a full brief with title, summary, problem statement, opportunity, solution directions, demand signals, competitive landscape, and source snapshots.

## Security

- CSP headers with strict `default-src 'self'`
- CORS restricted to production domain only
- Rate limiting: 10 ratings/min per session, 100 req/min per IP
- No PII collected — anonymous session-based ratings via httpOnly cookies
- All secrets in GCP Secret Manager
- Pydantic validation + SQLAlchemy parameterized queries
- `server-only` package prevents client-side DB imports
