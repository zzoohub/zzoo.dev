현대 웹 애플리케이션을 구축할 때 프레임워크 선택은 매우 중요합니다. 단순히 지금 인기 있는 것이 아니라, 최고의 개발자 경험과 최종 사용자 성능을 제공하는 것이 중요합니다. 저에게 그 답은 **Next.js**였습니다.

## 속도의 필요성

혼자서 일하는 팀에서는 라이브러리가 아닌 플랫폼이 필요했습니다. Next.js는 라우팅, SSG, 이미지 최적화, 배포를 하나의 패키지로 제공합니다.

블로그 게시물과 케이스 스터디에는 `next-mdx-remote`와 함께 **MDX**를 사용합니다. 마크다운에 React 컴포넌트를 직접 임베드할 수 있어 인터랙티브한 코드 예제와 아키텍처 다이어그램에 완벽합니다.

```tsx
// Next.js App Router로 간단한 정적 생성
export async function generateStaticParams() {
  const posts = await getAllPosts("ko");
  return posts.map((post) => ({ slug: post.slug }));
}
```

## 성능 최적화

App Router는 기본적으로 훌륭합니다. 정적 생성으로 1초 미만의 First Contentful Paint를 달성하는 것이 얼마나 쉬운지 정말 인상적이었습니다.

핵심은 초기 JavaScript 번들을 gzip 기준 100KB 미만으로 유지하는 것이었습니다. 서버 컴포넌트를 기본으로 사용하고 인터랙티비티가 정말 필요한 곳에서만 클라이언트 컴포넌트를 선택함으로써 번들은 가볍게 유지되었습니다.

## 결론

궁극적으로 최고의 도구는 자신의 필요에 맞는 도구입니다. 제 포트폴리오에는 Next.js가 모든 조건을 충족했습니다: 속도, 개발자 경험, 확장성.
